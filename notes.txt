OneToOne - в стране может быть один президент и президент может занимать эту должность только в одной стране.
То есть один уникальный объект пре надлежит другому уникальному объекту.

OneToMany - в одном департаменте могут работать несколько сотрудников. Но эти сотрудники не могут работать в других департаментах одновременно.
ManyToOne - перевернутое OneToMany
ManyToMany - в один кружок могут ходить несколько детей. Дети так же могут ходить в другие кружки.

Uni-direction - когда только одна сторона знает про отношение
Bi-direction - когда обе стороны знают про отношение

Каскады.
Выполнение операций не только на объекте над которым производят какие то изменения но и на связанных объектах с ним.
Не для всех задач подходит, нужно это учитывать.
ALL
PERSIST сохранение
MERGE объединение
REMOVE удаление
REFRESH обновление состояния объекта
DETACH

жизненный цикл Entity:
1)New. Пока что не ассоциирован с БД
2)Managed. Когда объект был ассоциирован с БД методом persist, save, get, update
3)Removed. Когда объект удалили.
4)detached. Когда объект открепили и он больше не ассоциирован с БД.

Persistence Context - это коллекция объектов которые в статусе Managed.
Если при обращении к БД объект есть в контексте то обращения к БД не будет, объект будет возвращен с контекста(кроме метода refresh.
Он всегда обращается в БД).

фабричный метод

mapped by указываем когда хотим создать бинарную связь вместо унарной, указываем имя поля другого класса с которым связались. Когда связка ManyToMany не используется mappedBy
Когда связываешь объекты по foreign key то указываешь целый объект с аннотацией  @JoinColumn(name="details_id")
где имя foreign key класса в котором объявляем или если есть @JoinTable то foreign key таблицы название которой в JoinTable

@JoinTable(name = "child_section" имя  join table
      , joinColumns = @JoinColumn(name = "child_id") имя столбца с которым связан объект
      , inverseJoinColumns =@JoinColumn(name = "section_id")) имя столбца через которое может достучаться к другой связке

если нужно разорвать связь объектов то монро просто засетить null.

можно указать в OneToMany @JoinColumn foreign key чужой таблицы когда создаться унарная связь

Eager загружает моментально связанные объекты
Lazy отложенная загрузка
Дефолтный тип выборки для  One в конце это Eager
